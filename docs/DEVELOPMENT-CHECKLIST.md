# Development Checklist

## Before Starting Development

- [ ] Read `PROJECT-CONTEXT.md`
- [ ] Read `AGENT-INSTRUCTIONS.md`
- [ ] Read `CROSS-DOMAIN-REFERENCE.md`
- [ ] Read domain-specific README
- [ ] Read function documentation
- [ ] Understand dependencies
- [ ] Check for existing implementations

## During Development

- [ ] Follow established patterns
- [ ] Use shared utilities (no duplication)
- [ ] Support department filtering
- [ ] Respect permissions
- [ ] Use primary count units
- [ ] Follow 2D inventory model (if applicable)
- [ ] Add error handling
- [ ] Add validation rules
- [ ] Write clean, maintainable code
- [ ] Add code comments for complex logic

## Before Committing

- [ ] Test functionality
- [ ] Check for errors
- [ ] Verify integration points
- [ ] Update documentation if needed
- [ ] Write clear commit message
- [ ] Follow commit message format

## Before Merging to Main

- [ ] All tests pass
- [ ] Code reviewed (if working with team)
- [ ] Documentation updated
- [ ] No merge conflicts
- [ ] Integration tested
- [ ] Performance acceptable
- [ ] Error handling complete
